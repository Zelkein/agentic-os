# Groupe CMI Voice & Tone

## Brand Personality

**Groupe CMI Experts-Conseils Inc.** is an engineering consultancy for MEP (Mechanical, Electrical, Plumbing) systems, architectural coordination, and energy efficiency.

### Core Voice Principles

- **Technical, not flowery** — Direct, specific, no jargon without explanation
- **Coordinator first, engineer second** — We solve collaboration problems, not just technical problems
- **Bias toward clarity** — Every decision documented, every assumption stated
- **Process-minded** — We build workflows that scale, not one-off solutions

### Tone for Different Audiences

**Internal (team):** Direct, concise, action-oriented
- "Coordination blocked on panel location. Ramy/Safa sync tomorrow 2pm."
- "Load calc complete. Headroom 20%. Ready for CAD."

**External (architects, clients):** Professional, confident, collaborative
- "MEP coordination phase complete. No conflicts identified. Ready for calculation phase."
- "Energy efficiency strategy: 15% reduction via geothermal + APH-Select controls."

**Documentation (process, workflows):** Structured, step-by-step, assumptionsexplicit
- Headings use imperative ("Enforce the coordination gate")
- Every action has a rationale
- Every assumption has a source

## Writing Rules

1. **No em dashes** — use commas or separate sentences
2. **Specific over general** — "ASHRAE 62.1 Table 6.2.2.1" not "ventilation standard"
3. **Action verbs lead** — "Validate coordination" not "Make sure coordination is validated"
4. **Sources cited** — Every calc assumption, every decision point
5. **Concrete examples** — "8 GPM peak demand" not "reasonable demand"

## Word Choices

| Use | Instead Of |
|-----|-----------|
| resolve | address, handle |
| enforce | ensure, make sure |
| validate | check, verify (unless it's validation logic) |
| blockers | issues, problems (when they block progress) |
| Phase gate | milestone, checkpoint |
| duct velocity | air speed, CFM |
| demand load | peak load (be specific) |
| fixture unit | count (use standard term) |

## Visual & Tone

- Numbers: Use units explicitly (°F, GPM, ft/min, Btu/hr)
- Dates: YYYY-MM-DD in logs, "May 14" in comms
- Status: ✓ complete, ⚠ note/caution, 🛑 blocker
- Structure: Checklists and tables over prose
